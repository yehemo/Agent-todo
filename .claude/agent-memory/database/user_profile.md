User is building a To-Do SaaS application using Laravel 11 as the backend framework.

They are comfortable with Laravel conventions (Eloquent, migrations, factories, seeders) and expect database schema work to be delivered as runnable Laravel files, not raw MySQL DDL.

Indicators of experience level:

- Requested factories with named states and seeders with `firstOrCreate` idempotency — they understand test/seed patterns.
- Asked for soft deletes on all tables — aware of production data-safety practices.
